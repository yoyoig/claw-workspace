## 审核清单

### 1. 完整性审核

- [ ] 执行摘要完整（500字以内，含核心结论）
- [ ] 六个章节齐全（背景、地缘、经济、军事、预测、建议）
- [ ] 附录完整（方法论、数据来源、团队）
- [ ] 所有支撑材料齐全

### 2. 一致性审核

- [ ] 各章节结论无矛盾
- [ ] 数据引用一致（不同章节同一数据）
- [ ] 预测概率与各分项分析一致
- [ ] 政策建议与分析结论对应

### 3. 准确性审核

- [ ] 关键数据有来源支撑
- [ ] 预测结论与 Phase 3 输出一致
- [ ] 引用信息与 Phase 1 采集一致
- [ ] 无明显事实错误

### 4. 可读性审核

- [ ] 执行摘要可在 15 分钟内把握要点
- [ ] 章节逻辑清晰
- [ ] 专业术语有解释
- [ ] 语言简洁专业

### 5. 可操作性审核

- [ ] 政策建议具体明确
- [ ] 建议有优先级
- [ ] 预警指标可观测
- [ ] 时间节点清晰

## 交付要求

### 必须交付

1. **审核报告** (`result.md`)
   - 审核结果（通过/需修订）
   - 问题清单（如有）
   - 修订建议（如有）
   - 最终确认签名

2. **发布版本** (`artifacts/final_report.md`)
   - 审核通过的最终报告
   - 如需修订，修订后的版本

3. **项目总结** (`artifacts/project_summary.md`)
   - 项目执行回顾
   - 团队贡献记录
   - 经验教训

### 质量标准

- 通过标准：所有审核清单项达标
- 需修订：1-3 项不达标，可快速修复
- 不通过：4+ 项不达标，需重大修订
